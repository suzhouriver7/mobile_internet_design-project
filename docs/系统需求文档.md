# 系统需求文档
## 1 项目可行性与需求分析
### 1.1 项目背景
在如今的大学生活中，同学们经常有约朋友伙伴一起出去进行**体育活动**,**聚餐**,**玩耍**的需求，但有时也有找不到合适的朋友一起出行的烦恼。
### 1.2 业务领域
校园学生生活
### 1.3 开发目标
我们的项目致力于开发一个**预约系统平台**,不管是吃喝玩乐，还是体育运动，只要你想约别人一起进行某项活动,just use it.
我们的应用分为**预约活动**,**动态**，**AI问询**三大功能。
在预约活动模块，你可以发布**订单**来发布自己想要预约的活动，时间，地点等等，别人可以通过问询和接受订单来匹配
在动态板块，你可以引用已完成的订单，来发布自己与有缘人在活动结束之后的心情、思考等等
我们也内置接入了ai,可以完成在动态发布之前修图，生成视频的功能，也可以与ai对话来获取你想了解的信息。
### 1.4 竞品分析
在北理工，最火的校园应用有贝塔驿站和bit101等等，但是他们都没有一个专注于预约系统的服务，在贝塔驿站发布一个帖子找一个朋友去打球可能会石沉大海———因为太多不相关的帖子充斥整个社区。我们的应用致力于解决这一有市场，有需求，大有可为的问题。
